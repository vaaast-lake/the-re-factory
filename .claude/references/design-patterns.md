-   **Creational** (생성 패턴 - 객체 생성 방식)
    -   **Factory Method** (팩토리 메서드: 서브클래스에서 생성할 객체 타입 결정)
    -   **Abstract Factory** (추상 팩토리: 관련 객체군을 생성하는 인터페이스)
    -   **Builder** (빌더: 복잡한 객체를 단계별로 생성)
    -   **Prototype** (프로토타입: 기존 객체를 복제하여 새 객체 생성)
    -   **Singleton** (싱글턴: 클래스의 인스턴스가 하나만 존재하도록 보장)
-   **Structural** (구조 패턴 - 클래스/객체 조합)
    -   **Adapter** (어댑터: 호환되지 않는 인터페이스를 연결)
    -   **Bridge** (브리지: 추상화와 구현을 분리하여 독립적 변경 가능)
    -   **Composite** (컴포지트: 개별 객체와 복합 객체를 동일하게 처리)
    -   **Decorator** (데코레이터: 객체에 동적으로 기능 추가)
    -   **Facade** (퍼사드: 복잡한 서브시스템에 간단한 인터페이스 제공)
    -   **Flyweight** (플라이웨이트: 많은 수의 유사 객체를 효율적으로 공유)
    -   **Proxy** (프록시: 다른 객체에 대한 접근을 제어하는 대리자)
-   **Behavioral** (행위 패턴 - 객체 간 책임 분배)
    -   **Chain of Responsibility** (책임 연쇄: 요청을 처리할 객체를 체인으로 연결)
    -   **Command** (커맨드: 요청을 객체로 캡슐화하여 매개변수화, 큐잉, 로깅)
    -   **Iterator** (반복자: 내부 구조를 노출하지 않고 순차 접근)
    -   **Mediator** (중재자: 객체 간 상호작용을 캡슐화하여 결합도 감소)
    -   **Memento** (메멘토: 객체의 상태를 저장하고 복원)
    -   **Observer** (옵저버: 객체 상태 변화를 관찰자들에게 자동 알림)
    -   **State** (상태: 객체의 상태에 따라 행동을 변경)
    -   **Strategy** (전략: 알고리즘군을 정의하고 교환 가능하게 만듦)
    -   **Template Method** (템플릿 메서드: 알고리즘 구조를 정의하고 일부 단계를 서브클래스에 위임)
    -   **Visitor** (방문자: 객체 구조에서 알고리즘을 분리)
