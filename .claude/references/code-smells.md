-   **Bloaters** (비대해진 코드)
    -   **Long Method** (긴 메서드: 너무 많은 일을 하는 메서드)
    -   **Large Class** (거대한 클래스: 너무 많은 책임을 가진 클래스)
    -   **Primitive Obsession** (기본 타입 집착: 객체 대신 기본 타입을 과도하게 사용)
    -   **Long Parameter List** (긴 매개변수 목록: 메서드가 너무 많은 매개변수를 받음)
    -   **Data Clumps** (데이터 뭉치: 항상 함께 나타나는 데이터 그룹)
-   **Object-Orientation Abusers** (객체지향 원칙 위반)
    -   **Switch Statements** (Switch문 남용: 다형성으로 대체 가능한 조건문)
    -   **Temporary Field** (임시 필드: 특정 상황에서만 사용되는 필드)
    -   **Refused Bequest** (거부된 유산: 상속받았지만 사용하지 않는 부모 기능)
    -   **Alternative Classes with Different Interfaces** (다른 인터페이스를 가진 대체 클래스: 같은 일을 하지만 메서드명이 다른 클래스들)
-   **Change Preventers** (변경 방해 요소)
    -   **Divergent Change** (발산적 변경: 한 클래스가 여러 이유로 변경됨)
    -   **Shotgun Surgery** (산탄총 수술: 한 가지 변경을 위해 여러 클래스를 수정해야 함)
    -   **Parallel Inheritance Hierarchies** (평행 상속 계층: 한 클래스 상속 시 다른 클래스도 함께 상속해야 함)
-   **Dispensables** (불필요한 요소)
    -   **Comments** (주석: 코드를 설명하기 위한 과도한 주석)
    -   **Duplicate Code** (중복 코드: 같은 코드가 여러 곳에 존재)
    -   **Lazy Class** (게으른 클래스: 충분한 역할을 하지 않는 클래스)
    -   **Data Class** (데이터 클래스: 필드와 getter/setter만 있는 클래스)
    -   **Dead Code** (죽은 코드: 사용되지 않는 코드)
    -   **Speculative Generality** (추측성 일반화: 미래를 위해 만든 불필요한 추상화)
-   **Couplers** (과도한 결합)
    -   **Feature Envy** (기능 선망: 다른 클래스의 데이터에 과도하게 접근)
    -   **Inappropriate Intimacy** (부적절한 친밀도: 클래스 간 지나친 의존)
    -   **Message Chains** (메시지 체인: a.getB().getC().getD() 형태의 연쇄 호출)
    -   **Middle Man** (중개자: 단순히 다른 객체에 위임만 하는 클래스)
